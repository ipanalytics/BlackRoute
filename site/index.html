Why teams use it

Reputation data you can explain, rebuild, and ship.

Commercial reputation APIs are useful, but not every gateway, proxy, fraud system, or enrichment job should depend on a live external lookup. Blackroute gives operators a repeatable local artifact that can be reviewed, diffed, versioned, and deployed where latency, privacy, or auditability matter.

Gateway policy

Use reputation labels as one signal for firewall, proxy, WAF, and edge access decisions.

Fraud checks

Enrich login, signup, payment, and abuse workflows without calling an external API at runtime.

SIEM enrichment

Attach threat and infrastructure context to IP events in pipelines, dashboards, and investigations.

Dataset engineering

Produce deterministic MMDB, CSV, JSONL, and stats files for internal data products.

Opinionated by design

Reputation compiler, not a verdict engine.

Blackroute does not resolve hostnames, query PTR records, crawl DNS, fingerprint networks, or scan anything. It downloads configured feeds, extracts public IP addresses and CIDR prefixes, attaches labels, merges duplicates, and writes deterministic output files. Use the result as a signal, combine it with allowlists and policy, and review high-impact blocking decisions before enforcement.

Record model

Built for explainable enforcement.

A lookup should not return a mysterious block/no-block answer. Blackroute keeps the matched prefix, source identifiers, confidence, threat labels, infrastructure labels, score, and build timestamps so decisions can be traced back to inputs.

Included source families

Public feeds, normalized into one schema.

blocklist.de SSH/mail/web/IMAP/FTP/SIP/bots Emerging Threats compromised and hostile hosts CINSscore multi-sensor risk FireHOL anonymizer and 1-day abuser feeds Spamhaus DROP and ASNDROP-derived infrastructure Team Cymru full bogon prefixes abuse.ch Feodo Tracker botnet C2 IPs SANS ISC DShield, GreenSnow, and IPsum

Commercial and API-key feeds are intentionally not bundled. Add private entries in configs/feeds.yaml when your license allows local redistribution or internal use.

Threat labels

Recent attack activity, bot signals, malware hosts, compromised systems, persistent attackers, and community or multi-sensor high-risk indicators.

Infrastructure labels

Bogons, anonymous infrastructure, cybercrime prefixes, aggregate anonymizers, and ASN-derived high-risk network context.

Quick start

Build locally or automate in CI.

Use the wrapper for normal operation, skip MMDB for data-only exports, or run a selected set of feeds for focused testing.

bash scripts/setup-server.sh
./run.sh
./run.sh --skip-mmdb
./run.sh --only=blocklist_de_ssh,emergingthreats_compromised
./run.sh --feeds=configs/feeds.yaml --output=release
